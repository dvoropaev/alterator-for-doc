3.2 Внутренние структуры данных

Alterator-manager хранит загруженные интерфейсы в хеш-таблице backends_data. Ключ — имя узла, значение — набор интерфейсов. Каждый интерфейс представлен структурой InterfaceObjectInfo, содержащей имя модуля, action id, описание интерфейса, таблицу методов, vtable и лимит потоков.1

Методы представлены структурами MethodObjectInfo с двумя вложенными таблицами: method_data (значения из backend-файла) и environment (переменные окружения). Такой подход позволяет хранить данные модуля executor и дополнительные настройки для других модулей.1

Диаграмма отношений

Структура данных организована вложенно: backends_data → interface → method → method_data/environment. Это обеспечивает быстрый доступ к информации о методе при обработке D-Bus вызова и позволяет модулям расширять собственные параметры без изменения ядра.1
