2.3 Описание сущности Backend и вспомогательных типов

Backend описывает D-Bus интерфейс, который реализует модуль executor. Файл содержит обязательные ключи module, interface, name и таблицу методов. Значение interface допускает короткое имя без точки — в этом случае добавляется префикс org.altlinux.alterator.1

Каждый метод объявляется в подтаблице [methods.<имя>], где определяются команда запуска, использование stdin/stdout, сигналы, ограничения на объём вывода и таймаут. Параметр thread_limit позволяет ограничить число параллельных вызовов.1

Диагностические инструменты

Сущности типа Diag описывают тесты, которые выполняются через backend. Они поддерживают локализованные имена, список тестов и атрибуты assignment, report_suffix. Сами тесты перечисляются в словаре tests и оформляются как набор параметров с отображаемыми именами и требованиями.12

Компоненты и редакции

Дополнительные типы Alterator Entry — Component и Edition — фиксируют списки пакетов, поддерживаемые архитектуры и секции отображения. Они применяются в инструментах управления комплектами поставки и наследуют общие правила именования и локализации.1
