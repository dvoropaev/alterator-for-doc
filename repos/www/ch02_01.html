2.1 Базовый формат и правила

Alterator Entry — подмножество TOML, предназначенное для описания сущностей Alterator. Файлы именуются по типу и получают расширения .application, .backend, .service и другие, соответствующие значению ключа Type.1

Каталоги размещения зависят от происхождения сущности: системные поставляются из /usr/share/alterator/<директория>, пользовательские и переопределения — из /etc/alterator/<директория>. Для backend предусмотрены режимы system и user с отдельными подкаталогами.1

Структура файла

  • Кодировка — UTF-8; строки воспринимаются регистрозависимыми.1
  • Комментарии — строки, начинающиеся с #, сохраняются при чтении и записи.1
  • Группы — объявляются как [groupname]; все пары ключ-значение до следующего заголовка относятся к группе.1
  • Записи — формат Key = Value; ключи регистрозависимы и используют символы A-Za-z0-9-.1
  • Типы значений — поддерживаются string, localestring, iconstring, stringlist, command, boolean, numeric; для массивов используется разделитель ;.1
  • Локализация — значения с постфиксом локали выбираются по правилам сопоставления LC_MESSAGES.1

Рекомендации по расширению

Дополнительные ключи следует оформлять с префиксом X-PRODUCT или выносить в отдельные группы, чтобы избежать конфликтов пространств имён.1
