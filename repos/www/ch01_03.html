1.3 Жизненный цикл управления сервисом

Интерфейс org.altlinux.alterator.service1 предоставляет полный набор методов для жизненного цикла сервиса: получение описания, развёртывание, конфигурация, запуск, остановка, резервное копирование, восстановление и снятие развёртывания.1

Пользовательский модуль alterator-manager-services отображает состояние сервиса (не развёрнут, развёрнут, запущен) и даёт доступ к действиям через кнопки «Развернуть», «Запустить», «Настроить» и вкладки «Параметры» и «Ресурсы».2

Порядок взаимодействия

  1. Получить описание сервиса через метод Info() для загрузки структуры параметров в интерфейсе.1
  2. Выполнить развёртывание методом Deploy(); интерфейс отобразит диалог настройки и соберёт параметры.12
  3. После развёртывания вызвать Start() для запуска системных юнитов или Configure() для изменения параметров.1
  4. При необходимости выполнить Backup() и Restore() для резервирования и восстановления конфигурации.1
  5. Использовать Status() для контроля состояния; коды 0, 127, 128 различают этапы жизненного цикла.12
  6. Завершить работу методом Stop() и Undeploy(), когда сервис необходимо отключить.1
