1.2 D-Bus топология и интерфейсы

Alterator-manager обрабатывает backend-файлы последовательно: системные каталоги просматриваются в фиксированном порядке, затем подключаются пользовательские. Для режима --user применяются отдельные пути. Каждый backend-файл описывает один объект и один интерфейс; дубликаты игнорируются.1

Backend хранит корневую таблицу с обязательными полями: type = "Backend", название модуля-обработчика, имя узла и имя интерфейса. Короткое имя интерфейса автоматически дополняется префиксом org.altlinux.alterator, что обеспечивает единообразное пространство имён на шине.1

Методы интерфейса описываются в разделе [methods.*]. Для модуля executor указывается команда, параметры, ограничения по потокам и дополнительные настройки ввода-вывода. Каждый метод маппится на команду оболочки, в которую подставляются аргументы из D-Bus вызова.12

Основные элементы топологии

  • Узлы — объектные пути формируются из значения name, дополненного префиксом /org/altlinux/alterator/.2
  • Интерфейсы — задаются полным или коротким именем; допускаются буквы, цифры и символ подчёркивания.1
  • Модули — указываются в ключе module. На текущий момент используется executor, который запускает внешние команды.2
  • Ограничения — параметры thread_limit, stdout_*_limit, timeout задают контроль параллелизма и ресурсов для каждого метода.2
