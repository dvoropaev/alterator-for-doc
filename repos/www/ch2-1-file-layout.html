2.1. Структура каталогов и именование

Файлы Alterator Entry именуются по шаблону <Name>.<Type>, где <Type> соответствует значению ключа Type в содержимом. Для типа diagnostictool допустимо сокращение .diag.Источник

Каталоги хранения разделены по режиму применения:

  • /usr/share/alterator/<категория>/ — системные сущности, устанавливаемые пакетами.Источник
  • /etc/alterator/<категория>/ — конфигурации и расширения, созданные администраторами.Источник
  • backends/user/ — пользовательские backend-файлы, применяемые в режиме --user.Источник

Наиболее употребимые каталоги:

Тип сущностиКаталог
Applicationapplications
Backendbackends
Categorycategories
Diagdiags
Objectobjects
Serviceservices
Componentcomponents

Соблюдение структуры гарантирует корректную загрузку описаний и минимизирует риск конфликтов при обновлениях.
