5.2 Параметры и ресурсы примера test_service1

Файл test_service1.service демонстрирует использование всех типов параметров. Он включает локализованные названия, комментарии и флаг enable_force_deploy, а также объявление диагностического инструмента.1

Параметры охватывают строки, числа, булевы значения, перечисления, массивы и составные объекты. Для каждого параметра указан контекст, обязательность и значение по умолчанию. Пример string_array_example использует регулярное выражение и ограничения на размер массива, а composite_param показывает структуру с дочерними свойствами и собственными значениями.1

Рекомендации по адаптации

  • Повторяйте шаблон контекстов, чтобы helper мог отличить параметры для deploy, configure и status.
  • Используйте required для обязательных полей в конкретных действиях и constant = true для скрытых технических значений.1
  • Для составных параметров обеспечьте локализацию дочерних полей и укажите границы числовых значений (пример param2 с min/max).
